## 5. Минимальный элемент деки за O(1)

Я использовал подход с двумя стеками для отслеживанием минимумов.

### Недостатки моей реализации

1. **Больше памяти** — 4 стека вместо 2 дек
2. **Требуется ребалансировка** — когда один стек пуст, O(n)
3. **Сложнее структура** — 4 стека vs 2 деки

Итого моя реализация отличается от рекомендации, но тоже обеспечивает O(1) для всех операций:

- `addFront()` — O(1)
- `addTail()` — O(1)
- `removeFront()` — O(1) амортизационная
- `removeTail()` — O(1) амортизационная
- `getMin()` — O(1)

## 1. Криптографические хэш-функции: SHA-256 и MD5

### Принципы построения

В основе криптографических хэш-функций лежит **итеративная последовательная схема**:

1. **Сжимающая функция** — преобразует k входных бит в n выходных
2. **Разбиение на блоки** — входной поток делится на блоки
3. **Итеративная обработка** — каждый блок объединяется с результатом предыдущей итерации
4. **Лавинный эффект** — каждый бит выхода зависит от всего входного потока

### Требования

| Требование | Описание |
|------------|----------|
| Сопротивление поиску прообраза | При наличии хеша h трудно найти m |
| Сопротивление второму прообразу | При наличии m₁ трудно найти m₂ с тем же хешем |
| Стойкость к коллизиям | Не должно быть эффективного алгоритма поиска коллизий |
| Псевдослучайность | Хеши должны быть непредсказуемы |
| Детерминированность | Одинаковый вход → одинаковый выход |

### MD5

| Параметр | Значение |
|----------|----------|
| Длина хеша | 128 бит |
| Структура | 4 цикла по 16 шагов (64 шага) |
| Статус | ❌ Взломан |

**Принцип работы:**
1. Данные дополняются до длины, кратной 512 бит
2. 4 регистра инициализируются начальными значениями
3. Данные обрабатываются блоками по 512 бит (4 раунда по 16 операций)
4. Результат = конкатенация 4 регистров

**Уязвимости:**
- 1996 г. — доказана возможность коллизий
- 2006 г. — коллизии находятся за **1 минуту**
- **Не рекомендован для использования**

### SHA-256

| Параметр | Значение |
|----------|----------|
| Длина хеша | 256 бит |
| Раундов | 64 |
| Статус | ✅ Стоек |

**Принцип работы:**
1. Сообщение дополняется до длины ≡ 448 (mod 512) + 64-битная длина
2. 8 начальных значений (константы из корней простых чисел)
3. 64 раунда с логическими функциями (CHOICE, MAJORITY)
4. Результат = конкатенация 8 регистров (256 бит)

**Преимущества:**
- ✅ Не найдено практических уязвимостей
- ✅ 2²⁵⁶ возможных значений
- ✅ Используется в Bitcoin, криптовалютах

### Сравнение

| Характеристика | MD5 | SHA-256 |
|----------------|-----|---------|
| Длина хеша | 128 бит | 256 бит |
| Стойкость к коллизиям | ❌ Взломан | ✅ Стоек |
| Скорость | Быстрее | Медленнее |
| Рекомендация | ❌ Не использовать | ✅ Рекомендуется |

### Лавинный эффект

| Вход | MD5 |
|------|-----|
| `bitcoin` | `0xCD5B1E4947E304476C788CD474FB579A` |
| `Bitcoin` | `0xD023EC040F79F1A9B2AC960B43785089` |

Изменение **одного символа** полностью меняет хеш.

### Вывод

Криптографические хэш-функции используют итеративную схему для достижения:
- ✅ Лавинного эффекта
- ✅ Стойкости к коллизиям (SHA-256)
- ✅ Псевдослучайности

**MD5 устарел**. **SHA-256** — стандарт для криптографических приложений.
